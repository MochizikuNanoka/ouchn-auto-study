# 控制面板界面优化说明

## 目标

将现有的深色浮动控制面板整理为更易扫读的学习控制台。保留墨黑、暖白与陶土橙的既有识别，不改变任何自动化、缓存、恢复或更新检查行为。

## 设计方向

- 以「暖石墨控制台」为视觉基调：深色底承载长时间阅读，陶土橙只用于主操作和运行状态。
- 标题采用中文衬线字体，正文保持清晰的中文无衬线字体；避免通用霓虹、渐变和装饰性图标。
- 将状态、操作、统计和日志拆成稳定分区，操作优先级依次为开始、暂停、停止、缓存重置、调试。

## 验收标准

1. 折叠、拖动、缩放、开始、暂停、停止、缓存重置、调试和更新检查的选择器与行为不变。
2. 面板在 320px 至 800px 宽度内不溢出；小屏时主操作仍可辨识。
3. 主操作、危险操作、禁用状态和键盘焦点均有明确视觉状态。
4. 日志保留原始级别颜色，并新增清晰的区域标题，不修改日志内容。
5. DEBUG 默认不输出；用户只能在“调试与更新”区域显式开启本脚本的 DEBUG 日志。
6. 运行现有 Node 回归、JavaScript 语法和 diff 空白检查。
