# 控制面板界面优化说明

## 目标

将现有的深色浮动控制面板整理为更易扫读的学习控制台。采用克制的 Apple 深色材质语言，不改变任何自动化、缓存、恢复或更新检查行为。

## 设计方向

- 以「深色玻璃材质」为视觉基调：半透明面板、轻量边框和层次阴影让它浮于学习页面之上，但不遮断页面操作。
- 使用浏览器系统字体与 Apple 系统语义色：蓝色用于主操作，绿色、黄色和红色分别表达成功、警告和错误；避免渐变、霓虹和装饰性图标。
- 将状态、操作、统计和日志拆成稳定分区，操作优先级依次为开始、暂停、停止、缓存重置、调试。
- 按下按钮即时缩放反馈；对“减少动态效果”“减少透明度”和“提高对比度”偏好提供等价的可访问性降级。

## 验收标准

1. 拖动、缩放、开始、暂停、停止、缓存重置、调试和更新检查的选择器与行为不变；折叠按钮初始状态与面板实际状态一致，首次点击即可折叠。
2. 面板在 320px 至 800px 宽度内不溢出；小屏时主操作仍可辨识。
3. 主操作、危险操作、禁用状态和键盘焦点均有明确视觉状态。
4. 日志保留原始级别颜色，并新增清晰的区域标题，不修改日志内容。
5. DEBUG 默认不输出；用户只能在“调试与更新”区域显式开启本脚本的 DEBUG 日志，并能通过辅助技术获知开关状态。
6. 运行现有 Node 回归、JavaScript 语法和 diff 空白检查。
